# Mainmenu Faction Fairy Card - 2026-06-30

## Frontier

Players have three selectable factions, and the first main-menu entry must show
the matching pixie/fairy and initial face expression for that faction.

## Static Evidence

- `layout_mainmenu.xml` binds the main-menu `pixie` component to
  `town_model.fairy_pose` and `town_model.fairy_face`.
- `_TownModel::init` copies the first `<mainmenu><infomation>` item into
  `TownModel +0x88/+0x8c`, exposed as `fairy_pose` / `fairy_face`.
- `_AnmPixie` constructor seeds three character ids:
  - country/faction slot 1 -> `117`
  - country/faction slot 2 -> `120`
  - country/faction slot 3 -> `111`
- `_AnmPixie::updateFairyImage(pose, face)` selects an adv character resource
  through the native resource helpers; XML does not directly name the file.
- `rule_resource.xml` player-select resources include base `adv_chara111`,
  `adv_chara117`, and `adv_chara120`.
- The local dump contains the adopted initial-expression resources:
- The local dump contains the adopted initial-expression resources:
  - `adv_chara117` and `adv_chara117_1_4`
  - `adv_chara120` and `adv_chara120_1_8`
  - `adv_chara111` and `adv_chara111_2_4`
- Static preview sheets were generated by decrypting the local `adv_chara*`
  resources with the known `k1` AES key, then composing labeled PNG sheets under
  `work/mainmenu-fairy-expression-preview-20260630/`.

## Adopted Mapping

```text
sword     -> country_id=1 -> fairyCharacterId=117 -> fairy_pose=1, fairy_face=4
technique -> country_id=2 -> fairyCharacterId=120 -> fairy_pose=1, fairy_face=8
magic     -> country_id=3 -> fairyCharacterId=111 -> fairy_pose=2, fairy_face=4
```

`fairyCharacterId` is server-side data only. The client-visible XML remains the
known parser shape:

```xml
<mainmenu>
  <infomation>
    <fairy_pose>...</fairy_pose>
    <fairy_face>...</fairy_face>
    <message>...</message>
  </infomation>
</mainmenu>
```

## Rejected / Not This Frontier

- Do not use `face_109/110/111` as the main-menu pixie selector. Those belong to
  the card/leader face resource chain, not this layout's `pixie` component.
- Do not add direct filename XML fields for the main-menu pixie; the native
  component selects resources from `country_id` slot plus `fairy_pose/face`.
- Do not reopen black main menu background, old face-black, BGM, voice, or
  clicked-dialogue-box work without new resource-miss or crash evidence.
